The Problem: Fragmentation & Inconsistency

Modern distributed systems promise agility but often deliver complexity. Teams struggle with:

Fragmentation

Disjointed tools (IaC, CI/CD, K8s), processes, and information across teams lead to toolchain sprawl, integration overhead, and slow feedback loops. (Learn more)

Inconsistency ("Snowflaking")

Manual tweaks and configuration drift create unique, hard-to-reproduce environments, hindering testing, deployment reliability, and scalability. (Learn more)

Ambiguity

Unclear dependencies, responsibilities, and configurations between services lead to miscommunication, rework, and integration issues. (Learn more)

The YAML/Container-Centric Trap

Traditional GitOps often focuses narrowly on container deployments via YAML, ignoring holistic environment needs, leading to brittle pipelines and organizational inefficiencies. (Learn more)

These issues often stem from "Scope Localization" - teams optimizing locally without a holistic, system-wide view enforced by the platform.

Our Solution: Holistic, Contract-Driven Environments

ONDEMANDENV provides a unified platform to manage the entire lifecycle of your distributed systems based on clear, codified contracts.

Contracts Library (`contractsLib`)

Define your system's architecture, service interfaces, dependencies, and deployment targets as code (TypeScript). This becomes the single source of truth, eliminating ambiguity. (Technical details)

Enver (Environment Version)

Manage complete, versioned, logical environments (infra, config, dependencies) as atomic units. Supports mutable (branch-based) and immutable (tag-based) envers for consistency and reproducibility. (Concept)

On-Demand Cloning

Instantly create high-fidelity, isolated clones of any enver (e.g., via a git commit comment). Perfect for development, testing, and experimentation without dependency conflicts. (Technical details)

Application-Centric Management

Treats the entire vertical slice (infra, runtime, security) as the application unit, managed cohesively, overcoming limitations of container-only approaches. (Learn more)

How It Works: Platform Overview

Define contracts, and the platform orchestrates infrastructure, dependencies, and deployments across your AWS accounts.

1. Define Contracts

Specify service interfaces, producers/consumers, and target environments (account/region) in the `contractsLib` repository.

2. Platform Orchestration

ODMD Central interprets `contractsLib`, updates the central config store, manages dependencies, and provisions required infra (IAM, pipelines via CDK).

3. Automated Lifecycle

Changes trigger automated builds, deployments, and updates for affected `Envers`. Cloning provides instant, isolated environments.

Implementation & Setup

ONDEMANDENV utilizes a multi-account AWS strategy and integrates tightly with GitHub. The reference implementation uses AWS CDK for Infrastructure as Code.

Typical setup involves a Central orchestration account and multiple Workspace accounts within an AWS Organization.
